Minutes of the 3-June-2003 CGS-WG telephone conference

A CGS-WG telecon was held on June 3, 2003 (8AM Pacific time). 

Attendees were:

Larry Flon (USC/ISI)

Carl Kesselman (USC/ISI)
Ellen Stokes (IBM)

Agenda

  1. Review latest JSIM draft
  2. Review Open Issues, and close as many as possible. See http://bugzilla.gridforum.org/cgs-wg for current issues.
  3. Any new business?
  1. Review Latest JSIM Draft

    The latest JSIM draft was posted on 2-June-2003, and is available in PDF here.  Some notes on the discussion:

    It was agreed that BatchService needs an InstanceID, but that in general, such classes are normal in CIM, even without having their own attributes.

    In JobQueue, it was agreed that MaxJobCPUTime should be a standard part of CIM 2.8, if it is still possible to get that done.  MaxRunningJob should be deleted, as it does not seem to have a valid use case.

    In QueueStatisticalData, it was agreed that at least one of RunningJobs/WaitingJobs, perhaps both, should be a standard part of CIM 2.8 if possible, but we need clarification first, since if you have one of these, it seems possible to derive the other using NumberOnQueue which is standard in 2.8.  So what is standard CIM policy in terms of representing derivable values?

    We agreed that we should start work on an XML schema (XSD) to represent the JSIM, but that we should not try to promote that as a standard at this time, but to publish it as a "strawman" and, outside of GGF, encourage implementations around it.
